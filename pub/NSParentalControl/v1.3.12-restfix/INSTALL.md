# NSParentalControl 单次时长跨用户串台修复（v1.3.12，基于 v1.3.11）

## 本次修复：用户 A 日用满后，用户 B 提前被「需要休息」锁定

### 现象（用户报告）
两个用户，日限额均 30 分钟、单次 15 分钟、休息 30 分钟。用户 A 全天用满后，
用户 B 刚开始玩（甚至才玩几分钟）就被提示需要休息/锁定。

### 根因（坑 11）
`sessionElapsedMin_` 是 Monitor 的**全局成员**，表示「整机连续游玩」已累计分钟数，
其重置点只有两处：触发强制休息时、休息结束时。

而每日额度耗尽那条分支（`remainingTimeInMinutes <= 0`）**没有清零它**。
于是用户 A 日用满（走日限额分支、未走休息分支）→ `sessionElapsedMin_` 残留 ~15 →
这个值随全局计数串到用户 B → B 还没玩满 15 分钟就撞上 `sessionLimit` → 被误判强制休息。

> 注：日限额本身是**按 uid 各自计算**的（正确）；出问题的只是「单次已玩时长」这个全局计数。

### 为什么不在「切账号 / 关游戏」时清零（设计取舍）
- 若「uid 一变就归零」：小孩可来回切账号把单次计数反复清零，从而**绕开**「玩 15 分休息 30 分」的强制休息。
- 若「关游戏就归零」：小孩玩 14 分→退回 HOME 关掉→再开→计数清零，是**更easy的绕过**。
- 因此保持「整机连续游玩」语义：A 玩 14 分切到 B、B 再玩 1 分，仍累计到 15 → 强制休息，
  **切账号 / 关游戏都不能白嫖休息**；日限额仍按 uid 各自封顶，不会“一直可以玩”。

### 正确做法（v1.3.12）
仅在**日限额耗尽**分支补一行清零，断掉跨用户串台：
```cpp
if(remainingTimeInMinutes <= 0) {
    sessionElapsedMin_ = 0;   // 坑11: 防止残留值串到下一个用户，误触发其强制休息
    NotificationsController::notifyTimeExpired();
    terminateCurrentApplication();
}
```
修复后：用户 A 日用满 → 计数清零 → 用户 B 从 0 开始，玩满 15 分才休息（符合预期）。
